Introduccion

Este documento presenta el diseño de una base de datos para una institución educativa. Se definen entidades como niños, padres, madres, responsables, educadores y aulas, junto con sus atributos y relaciones. Se proporcionan datos de muestra para ilustrar el funcionamiento del sistema.

Entidades

Entidad Niño

Atributo Descripción
id_nino (PK) Identificador único del niño
nombre Nombre del niño
primer_apellido Primer apellido del niño
segundo_apellido Segundo apellido del niño
fecha_nacimiento Fecha de nacimiento del niño
id_padre (FK) Identificador del padre (Foreign Key)
id_madre (FK) Identificador de la madre (Foreign Key)
id_responsable (FK) Identificador del responsable (FK)
domicilio Domicilio del niño
telefono1 Primer número de teléfono
comentario_telefono1 Comentario sobre el primer teléfono
telefono2 Segundo número de teléfono
comentario_telefono2 Comentario sobre el segundo teléfono
horario_asistencia Horario de asistencia
tarifa_mensual Tarifa mensual
incidencias_sociales Incidencias sociales
datos_clinicos Datos clínicos
comentarios_adicionales Comentarios adicionales
id_aula (FK) Identificador del aula (FK)
estado Estado del niño (activo, baja_temporal, baja)

Atributo Descripción

Atributo Descripción

Asociaciones

  • Niño - Padre: 1:N (un niño tiene un padre)
  • Niño - Madre: 1:N (un niño tiene una madre)
  • Niño - Responsable: 1:N (un niño tiene un responsable)
  • Niño - Aula: N:M (un niño puede estar en un aula, un aula puede tener varios niños)
  • Educador - Aula: N:M (un educador puede trabajar en un aula, un aula puede tener varios educadores)
  • Niño - Nivel: 1:N (un niño está en un nivel)
